# SSL Checker Telegram Bot

Telegram-бот на Go выполняет разовую и периодическую проверку TLS/SSL сертификатов сайтов.
Он подключается к указанному URL или hostname, проверяет доверенную цепочку системным
хранилищем корневых сертификатов и возвращает сведения о leaf-сертификате, SAN,
issuer, сроках действия, fingerprint, статусе отзыва по CRL, протоколе TLS, cipher suite и цепочке.

## Возможности

- `/check <url|host[:port]>` — разовая проверка сертификата.
- `/watch <url|host[:port]> [минуты]` — периодическая проверка по таймеру; успешные проверки не отправляют подробный отчет, бот присылает alert только при ошибке или предупреждениях.
- `/list` — список активных таймеров в текущем чате с интервалом для каждого host.
- `/unwatch <url|host[:port]>` — удаление таймера.
- Предупреждения об истечении, отсутствии SAN, несовпадении hostname, неполной цепочке и отзыве сертификата через CRL.
- Нет внешних Go-зависимостей: бот использует стандартную библиотеку и Telegram Bot API через long polling.

## Запуск

```bash
export TELEGRAM_BOT_TOKEN="<token from BotFather>"
./go-ssl-check
```

Скрипт `go-ssl-check` сам соберет бинарник в `.bin/go-ssl-checker-bot` и запустит бота. Список периодических проверок сохраняется в пользовательской config-директории (`$XDG_CONFIG_HOME/ssl-checker/watchers.json` или аналог ОС), поэтому после перезапуска бота таймеры восстанавливаются автоматически даже если директория проекта read-only. Скрипт по умолчанию задает `WATCHERS_FILE=/var/tmp/go-ssl-check/watchers.json`. Путь можно переопределить переменной `WATCHERS_FILE`; при старте бот выбирает один доступный для записи файл хранения (сначала существующий writable файл, затем config-директория, затем стабильный fallback `/var/tmp/go-ssl-check/watchers.json`) и дальше читает и пишет только его, чтобы `/list` не пустел после перезапуска.
Для ручной проверки перед запуском можно выполнить:

```bash
go test ./...
go build -o /tmp/ssl-checker-bot .
```

По умолчанию `/watch` запускает проверку раз в 24 часа. Минимальный интервал — 1 минута. При успешной периодической проверке бот молчит, чтобы не засорять чат; подробный alert отправляется только при ошибке подключения/валидации или найденных проблемах сертификата, включая обнаруженный отзыв сертификата.
